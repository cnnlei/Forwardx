# DDNS 和故障转移

DDNS 可以在服务器 IP 变化时自动更新域名解析，也可以用于入口组和转发组故障切换。

支持的常见服务商：

- Cloudflare。
- 华为云。
- 阿里云。
- 腾讯云 DNSPod。
- Webhook。

## 记录类型怎么选

| 类型 | 要求 |
| --- | --- |
| A | 成员机器需要有 IPv4 |
| AAAA | 成员机器需要有 IPv6 |
| CNAME | 成员机器需要配置 DDNS 域名 |

如果你选择 AAAA，但成员机器没有 IPv6，就不适合使用该记录类型。

## 故障切换建议

- 不要把故障切换时间设置得太短。
- 网络偶发抖动时，太短的时间可能导致频繁切换。
- 重要业务建议开启链路测试和延迟观察。
- 恢复后是否切回，取决于你是否希望优先使用主入口。

## 域名来源优先级

ForwardX 展示入口地址时，一般会优先使用用户在主机或组内填写的域名或 IP。

如果没有填写，会尝试使用自动检测到的公网地址或 DDNS 地址。

建议重要业务手动设置清晰的域名，避免用户直接依赖可能变化的 IP。
