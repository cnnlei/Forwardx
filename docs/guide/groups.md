# 转发组、入口组和出口组

## 转发组

转发组适合多个入口同时提供同一个服务，并通过 DDNS 做故障转移。

例如：

```text
入口 A -> 目标服务
入口 B -> 目标服务
入口 C -> 目标服务
```

当入口 A 故障时，可以切换到入口 B 或入口 C。

## 入口组

入口组用于把多台入口服务器收敛成一个入口资源。

适合：

- 多入口统一管理。
- 配合 DDNS 使用同一个域名。
- 给隧道或端口转发链复用。

## 出口组

出口组用于把多个出口服务器作为一个出口资源。

适合：

- 多落地出口。
- 出口故障切换。
- 隧道出口负载均衡。

## 命名建议

建议使用清晰名称：

```text
香港入口组
日本落地出口组
游戏业务转发组
```

组名称清楚，后续配置转发规则时就不容易把入口和出口选反。
