# 首次初始化

浏览器打开面板地址：

```text
http://服务器IP:3000
```

首次访问会进入初始化流程。

## 选择数据库

ForwardX 支持三种数据库：

| 数据库 | 适合谁 | 需要提前准备 |
| --- | --- | --- |
| SQLite | 新手、小规模、单机使用 | 不需要，面板会使用本地数据文件 |
| MySQL | 生产环境、长期使用、多用户 | MySQL 8.0.13 或以上版本、账号和数据库 |
| PostgreSQL | 已经有 PostgreSQL 环境的用户 | PostgreSQL 12 或以上版本、账号和数据库 |

建议：

- 不确定怎么选，就先选 SQLite。
- 后续用户变多、规则变多，可以再迁移到 MySQL 或 PostgreSQL。
- 如果你已经有现成数据库服务器，可以直接选择 MySQL 或 PostgreSQL。

SQLite 不需要额外填写数据库地址。MySQL 和 PostgreSQL 需要填写：

- 数据库地址。
- 端口。
- 用户名。
- 密码。
- 数据库名。
- 是否启用 SSL。

填写后需要先测试连接，连接成功后才能继续。

::: tip Docker 用户
如果面板通过 Docker 部署，数据库地址要填写“面板容器内部能访问到的地址”。例如数据库在同一个 compose 网络中时，通常填写数据库服务名；数据库在宿主机上时，通常填写宿主机内网 IP。不要直接把 `127.0.0.1` 当作宿主机数据库地址。
:::

::: warning 连接失败怎么看
如果容器日志出现 `getaddrinfo ENOTFOUND 数据库地址`，说明面板容器无法解析这个数据库主机名。请检查数据库容器和面板容器是否在同一 Docker 网络，或改用面板服务器能访问到的 IP/域名。
:::

## 创建管理员

数据库设置完成后，创建第一个管理员账号。

管理员可以：

- 管理用户。
- 管理主机。
- 管理转发规则。
- 管理链路、隧道、转发组。
- 配置系统设置、Telegram、邮件、DDNS。
- 执行面板和 Agent 升级。

## 设置面板公开地址

登录后建议第一时间进入：

```text
系统设置 -> 系统信息
```

填写面板公开地址。

如果没有域名：

```text
http://服务器IP:3000
```

如果已经配置了 HTTPS 域名：

```text
https://panel.example.com
```

::: warning 重要
面板公开地址会影响 Agent 安装、Agent 升级和 Agent 通讯。后续如果面板 IP 或域名变化，应及时修改。
:::
