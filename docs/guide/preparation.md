# 部署前准备

## 面板服务器

面板服务器是运行 ForwardX 后台和网页的机器。

| 项目 | 建议 |
| --- | --- |
| 系统 | Linux |
| CPU | 1 核起步，推荐 2 核以上 |
| 内存 | 1 GB 起步，推荐 2 GB 以上 |
| 磁盘 | 10 GB 以上 |
| 端口 | 默认 `3000` |

如果你要用域名访问面板，建议准备一个域名：

```text
panel.example.com
```

生产使用时推荐通过 Nginx、Caddy、宝塔等反向代理到面板端口，并开启 HTTPS。

## 被管理服务器

被管理服务器是安装 ForwardX Agent 的机器，也就是实际执行转发、隧道、测速和状态上报的机器。

要求：

- Linux 系统。
- 有 root 权限。
- 能访问面板地址。
- 如需 IPv6 转发，机器本身需要有可用 IPv6。
- 如需开放入口端口，需要服务器安全组和系统防火墙放行对应端口。

## 端口规划

| 端口 | 用途 |
| --- | --- |
| `3000` | 面板默认访问端口 |
| `80/443` | 使用域名和 HTTPS 反向代理时使用 |
| 自定义入口端口 | 用户业务访问端口 |
| 隧道监听端口 | GOST 或 ForwardX 隧道通讯 |

::: warning 注意
服务器云厂商安全组、系统防火墙和面板规则都可能影响端口连通。转发不通时，不要只看面板配置，也要检查安全组是否放行。
:::
