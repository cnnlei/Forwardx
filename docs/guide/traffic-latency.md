# 流量和延迟图表

ForwardX 会展示：

- 规则流量。
- 主机流量。
- 近 24 小时流量。
- 累计流量。
- 服务延迟。
- 链路自测延迟。
- 多跳链路每段延迟。

## 流量统计怎么看

流量统计用于判断规则是否真的有访问，也可以用于用户计费和额度管理。

如果某条规则流量不增长，建议检查：

- 规则是否真的有访问。
- 转发方式是否支持当前统计。
- iptables 或 nftables 规则是否存在。
- 入口端口是否正确。
- 目标地址和目标端口是否正确。

## 延迟图表怎么看

延迟图表用于观察服务质量和网络波动。

如果图表看起来波动较大，可以开启削峰或平滑显示。这个功能只影响页面展示，不会修改真实入库数据。

## 链路测试

链路测试用于判断入口、中转、出口和目标哪一段出现问题。

多级链路时不要只看入口到出口的总延迟，要重点看每段延迟和失败位置。
